#완주하지 못한 선수
https://school.programmers.co.kr/learn/courses/30/lessons/42576
##풀이 아이디어
해시맵 사용, 참가자 이름과 완주자를 저장한다, 두 값을 비교하여 없거나 같지않으면 값을 리턴한다.
## 시간 복잡도
배열을 여러 번 순회함으로 O(n)
##느낀점
HashMap 함수를 사용하는데 getOrDefault라는 함수가 중복된 값을 처리하는데 효율적으로 처리가 되었음.
null값을 생각 못했는데 예외처리를 하는 과정에서 발견하여 for문 처음에 걸러서 완성함.
if문안에서 두개를 비교할때 == 비교연산자를 사용했는데 ==의경우 주소값의 비교이기 때문에 값을 비교하는 equals를 사용해야한다.
해시맵을 하나만 사용하여 간결하게 문제풀이를 완수하는 경우도 확인함(다음 for문에서 값을 -1을하여 0이 아닌 값을 리턴)
