##풀이 아이디어
-이중 반복문을 사용 첫번째 배열의 값(i)과 두번째 배열의 값(j)을 비교한다.

##시간 복잡도
-O(n^2)
-개선후 O(n log n)


##느낀점
-값을 비교하는데 배열의 값을 전부 비교해야함으로 비효율적이고 시간또한 급격히 늘어남

-접두사의 경우 정렬을 하게 되면 해당문자는 반드시 다음값으로 나와야하기 때문에 시간과 코드의 가독성이 늘어남

-먼저 sort를 사용하여 정렬을 먼저하고 for문을 사용하여 결과를 도출한다.

-마지막 배열은 검사할 필요가 없음으로 배열의 길이는 -1, startsWith함수는 앞 문자열로 시작하면 true아니면 false를 리턴.

-endsWith 끝부분 검사, contains 포함되어 있는가
