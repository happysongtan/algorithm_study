## 풀이 문제
- [주소](https://leetcode.com/problems/combinations/)

## 풀이 아이디어
- 매번 1부터 탐색하지 않고, start 값을 기준으로 다음 숫자부터 탐색한다.
- 이미 선택한 숫자보다 큰 숫자만 선택하게 만들어 중복 조합을 방지한다.

## 시간 복잡도
- O(C(n, k) * k)
- 가능한 조합의 개수는 C(n, k)개이고, 각 조합을 answer에 저장할 때 길이 k만큼 복사한다.

## 느낀점
-중복제거를 set함수 방향으로 잡았지만 방향을 중복 입력을 방지하는 방향으로 바꿈
