## 풀이 문제
- [주소](https://www.acmicpc.net/problem/17298)

## 풀이 아이디어
-스택에 인덱스를 저장하고 배열을 순회
-현재 값이 스택의 top인덱스가 가리키는 값보다 클 경우. 해당 인덱스의 오큰수를 현재 값을 갱신
-이를 스택이 빌 때까지 반복함.

## 시간 복잡도
-O(N)

## 느낀점
-스택을 이용해서 문제를 풀었음
-기존에는 split을 사용했지만 StringTokenizer 사용해서 더 간편하게 만들었음
- ArrayDeque를 스택처럼 사용할 때는 add()보다 push()를 사용하는 것이 더 명확하다는 것을 알게 되었다.
- add는 뒤에 추가 push는 앞에 추가 (스택처럼 쓸때는 push pop, 큐는 add,poll)
