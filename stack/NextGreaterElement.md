##풀이 문제
- [주소](https://www.acmicpc.net/problem/17298)

##풀이 아이디어
-스택에 인덱스를 저장하고 배열을 순회<br>
-현재 값이 스택의 top인덱스가 가리키는 값보다 클 경우. 해당 인덱스의 오큰수를 현재 값을 갱신<br>
-이를 스택이 빌 때까지 반복함.<br>
##시간 복잡도
-O(N)

##느낀점
-이중 배열로 접근했으나, 시간 복잡도의 효율성에서 부족했다.<br>
-스택을 인덱스를 넣는 방향으로 전환하고 for내부에서 비교할때 값이 안정해진 인덱스를 비교할때 사용하였다<br>
