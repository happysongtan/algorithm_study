##풀이 문제
- 주소


##풀이 아이디어
- 입력된 배열을 정렬
- 각 숫자를 target으로 잡고, 투 포인터(start, end)를 이용해 두 수의 합이 target이 되는지 탐색한다.
- 합이 target보다 작으면 start 증가, 크면 end 감소 방식으로 탐색한다.
- start 또는 end가 target 인덱스와 같으면 건너뛴다.
- 
##시간 복잡도
- O(N^2)

##느낀점
- 처음에는 target을 3부터 시작하고 end를 target-1로 두었는데, 이 방식은 target 뒤쪽의 수를 사용할 수 없어 음수 케이스에서 틀릴 수 있다는 걸 알게 됐다.
- 이 문제는 자기 자신만 제외하고 배열 전체에서 두 수를 골라야 하므로, start=0, end=length-1로 두고 target 인덱스만 건너뛰어야 한다.
