#풀이 문제
- [주소](https://school.programmers.co.kr/learn/courses/30/lessons/42628)

##풀이 아이디어
-최댓값과 최솟값을 동시에 관리하면서 삭제해야 하므로 우선순위 큐를 Asc와 Desc로 분리해서 사용
-각각의 큐에서 삭제가 발생하면 반대쪽에서 바로 삭제를 하지않고 값을 저장하여 추후에 삭제를 함(Lazy Deletion)
-들어오는 문자열을 배열로 받으며 split를 사용 공백을 기준으로 두개의 값으로 배열에 넣음
-삭제를 할 때 값에 따른 삭제 추가의 편의성을 위해서 hashMap을 사용 key를 삭제할값 value를 삭제할값의 횟수를 저장하여 사용
-삭제 및 추가를 하기전에 항상 isEmpty로 비어있는 queue를 검사하지 않도록 방지
-index에 따른 연산 종료 후 남아 있는 삭제 queue의 값을 한 번 더 검사


##시간 복잡도
-O(n log n)

##느낀점
-문자열을 기준으로 입력값을 받을 때 처음 조건을 "D 1".equals 를 사용하였는데 추후 다른 문제나 뒤에 나올 코드에 직관성을 위해서 split로 배열로 분리후 사용하였음
-queue를 삭제할때 poll과 HashMap.put을 같이 사용하여 내부적으로 로직은 맞았으나 처음 로직을 작성할때와 계산 검증을 할때 직관성 및 안정성이 떨어져 각각 변수로 queue를 분리해 사용
-Desc와 Asc를 분리해서 사용하였는데 코드를 구성하는 과정에서 기준점을 가끔씩 놓쳐서 코드를 완성하는데 시간이 오래걸림.
