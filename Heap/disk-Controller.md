#풀이 문제
- [주소](https://school.programmers.co.kr/learn/courses/30/lessons/42627)

##풀이 아이디어
-jobs를 요청시간 기준으로 정렬을 한다.

-우선순위 큐를 활용하여 문제 내부에서 중요한 순서를 토대로 정렬 기준을 세워둔다.

-while문을 사용하여 아직 처리하지 않은 작업이 남아 있거나 큐에 대기중인 작업이 남아있는 동안 반복을 조건으로 걸어둠

-queue를 빼낼때 배열을 선언하여 값을 자유롭게 사용

##시간 복잡도
-O(N log N)

##느낀점
-우선순위 큐를 선언할 때 sort에 (a,b)-> a-b;를 했을때처럼 각 요소의 우선도를 선정하여 구조를 선언할 수 있음을 배움

-while문안에 조건을 넣을때 queue.isEmpty() or index<jobs.length이렇게 선언했는데 왼쪽 조건을 먼저 읽으면 원하는 기준을 충족하지 못하고 값이 튈 수가 있음을 배움

-queue.poll을 할때 queue안에 값이 두개이상일때 다루기 까다로웠는데 배열을 먼저 선언하면 이후 여러 조건에서도 자유롭게 queue.poll을 한뒤에도 그전에 값을 다룰 수 있게 됨
